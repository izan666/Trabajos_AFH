Índice

1. ¿Qué es el pre-entrenamiento?

El pre-entrenamiento en Inteligencia Artificial es la fase inicial en la que un modelo aprende patrones generales a partir de una gran cantidad de datos antes de ser ajustado para una tarea específica.

En modelos de lenguaje, el pre-entrenamiento consiste en exponer al modelo a grandes textos para que aprenda:

Gramática

Estructura del lenguaje

Relaciones entre palabras

Contexto y coherencia textual

Por ejemplo, modelos como los desarrollados por OpenAI o Google se pre-entrenan con enormes cantidades de texto antes de usarse para tareas concretas como responder preguntas o resumir información.

En nuestro caso, el objetivo fue pre-entrenar un modelo con el texto completo de Don Quijote de la Mancha para que aprendiera información específica sobre sus personajes y estilo.

2. Preparación del entorno

Antes de empezar el pre-entrenamiento, necesitaba preparar los datos correctamente.

Primero, accedí a Project Gutenberg y descargué la versión en texto plano (Plain Text UTF-8) del libro. Este formato es importante porque permite trabajar directamente con texto sin formato.

Sin embargo, el archivo original contenía:

Cabeceras

Notas legales

Marcadores de inicio y fin

Texto adicional que no forma parte de la novela

Si entrenaba el modelo con todo eso, podría aprender información irrelevante o generar errores.

Por eso, ejecuté un script propio que limpiaba el texto:

Detecté los marcadores de inicio y fin del libro.

Eliminé encabezados y pies de página.

Reduje saltos de línea innecesarios.

Guardé el texto limpio en un nuevo archivo listo para entrenar.

Este paso es fundamental porque en IA la calidad de los datos influye directamente en el resultado final.

3. Pasos que seguí para realizar el pre-entrenamiento

A continuación explico el proceso tal y como lo realicé:

Paso 1: Elegir un modelo base

Decidí utilizar un modelo ya existente en lugar de entrenar uno desde cero. En concreto, utilicé un modelo basado en GPT-2 en español disponible en la librería Hugging Face Transformers.

Esto me permitió partir de un modelo que ya entendía español y adaptarlo al contenido del Quijote.

Paso 2: Cargar el texto limpio

Después de limpiar el archivo, lo cargué como dataset de entrenamiento.

Dividí el texto en bloques de tamaño reducido (block_size=128) para que el modelo pudiera procesarlo correctamente sin sobrecargar la memoria.

Paso 3: Configurar el entrenamiento

Definí los parámetros principales:

Número de épocas (3)

Tamaño de batch pequeño (para poder ejecutarlo en CPU)

Carpeta donde se guardaría el modelo

Límite de checkpoints guardados

También verifiqué que las versiones de las librerías fueran compatibles antes de empezar, para evitar errores durante la ejecución.

Paso 4: Ejecutar el entrenamiento

Una vez configurado todo, inicié el entrenamiento.

En esta fase, el modelo:

Lee el texto.

Intenta predecir la siguiente palabra.

Ajusta sus pesos internos.

Reduce progresivamente el error.

Este proceso puede tardar varios minutos o incluso horas dependiendo del hardware disponible.

Paso 5: Guardar el modelo entrenado

Una vez terminado el entrenamiento, guardé:

El modelo ajustado.

El tokenizador.

Los archivos de configuración.

De esta forma, pude reutilizar el modelo sin necesidad de volver a entrenarlo.

Paso 6: Demostración

4. Resultados del pre-entrenamiento

Tras finalizar el proceso, probé el modelo haciéndole preguntas relacionadas con el libro.

Le pregunté quién era Sancho y quién era Dulcinea.

Como el modelo había sido entrenado con el texto completo, fue capaz de generar respuestas coherentes relacionadas con los personajes de la novela.

Esto demuestra que el modelo había internalizado información del texto durante el pre-entrenamiento.

El objetivo no era crear un modelo perfecto, sino comprobar que el proceso de adaptación había funcionado correctamente.

5. Ventajas del pre-entrenamiento

Permite reutilizar modelos grandes ya entrenados.

Reduce tiempo y coste computacional.

Mejora el rendimiento en tareas específicas.

Facilita la especialización temática (como en este caso con literatura clásica).

6. Limitaciones

Depende mucho de la calidad de los datos.

Puede generar errores si el texto no está bien limpio.

No garantiza comprensión real, solo patrones estadísticos.

Requiere recursos computacionales.

7. Conclusión

El pre-entrenamiento es una fase fundamental en el desarrollo de modelos de lenguaje. Permite que una red neuronal aprenda patrones generales y después pueda adaptarse a tareas o dominios concretos.

En este proyecto, partí de un modelo base en español y lo ajusté utilizando el texto limpio del Quijote. Tras preparar los datos, configurar el entrenamiento y ejecutarlo correctamente, el modelo fue capaz de generar respuestas coherentes sobre los personajes principales.

Este proceso demuestra cómo, mediante técnicas relativamente accesibles y herramientas actuales, es posible especializar modelos de lenguaje en contenidos específicos.
