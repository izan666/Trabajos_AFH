1. ¿Qué es un agente en Inteligencia Artificial?

Un agente de Inteligencia Artificial es un sistema que percibe su entorno, toma decisiones y ejecuta acciones de forma autónoma para cumplir un objetivo.

En el contexto de modelos de lenguaje, un agente no es solo el modelo en sí, sino el sistema que:

Recibe instrucciones (prompts)

Decide cómo procesarlas

Puede interactuar con herramientas externas

Devuelve una respuesta estructurada

Por ejemplo, cuando usamos sistemas desarrollados por OpenAI, normalmente interactuamos con un modelo en la nube. En este trabajo, en lugar de usar un servicio externo, configuramos nuestro propio agente funcionando localmente.

El objetivo fue crear un agente que funcionara en nuestro ordenador, utilizando nuestra propia tarjeta gráfica, sin depender de servidores externos.

2. Selección del modelo y entorno de realización

Para esta práctica decidí utilizar el modelo Qwen 2.5 Coder 3B Instruct, ya que está optimizado para tareas de programación y tiene un tamaño moderado (3B parámetros), lo que permite ejecutarlo en hardware limitado.

Elegí este modelo principalmente porque:

Está orientado a generación de código.

No es excesivamente pesado.

Funciona bien con recursos limitados.

En el entorno físico disponíamos de una tarjeta gráfica NVIDIA 3050 Ti con 4GB de VRAM, por lo que necesitaba un modelo ligero que no excediera esa memoria.

Para ejecutar el modelo localmente utilicé LM Studio, ya que permite:

Descargar modelos.

Ejecutarlos en local.

Levantar un servidor compatible con la API de OpenAI.

3. Configuración de LM Studio (Proceso realizado)

Para maximizar el rendimiento de la GPU sin superar los 4GB de VRAM, configuré el modelo de la siguiente manera:

Preset: Qwen2.5-Coder-3B-Instruct

Context Length: 12288 tokens

GPU Offload: MAX

Server Port: 1234

Decidí usar un contexto de 12288 tokens porque durante las pruebas anteriores había tenido errores cuando el contexto era demasiado bajo. De esta forma evité fallos en conversaciones un poco más largas.

También configuré la carga en GPU al máximo para evitar usar memoria RAM, ya que la RAM es aproximadamente 20 veces más lenta que la VRAM.

Finalmente, activé el servidor local en el puerto 1234, que sería el punto de conexión para el agente.

4. Integración con OpenCode (Proceso realizado)

Para que el agente funcionara correctamente, necesitaba redirigir las peticiones.

Por defecto, aplicaciones como OpenCode están configuradas para conectarse a la API oficial de OpenAI en la nube. Lo que hice fue modificar esa configuración.

Creé un archivo opencode.json donde cambié el endpoint:

En lugar de apuntar a internet, lo redirigí a:

http://localhost:1234

De esta manera:

OpenCode cree que está hablando con la API de OpenAI.

En realidad, las peticiones se procesan en mi propio ordenador.

LM Studio actúa como servidor que imita la estructura de respuestas de OpenAI.

Una vez configurado todo correctamente, al abrir OpenCode pude seleccionar mi modelo (renombrado como "qwen 3B coder") y empezar a interactuar con él como si fuera un modelo en la nube.

5. Pruebas interactivas

Una vez integrado el agente, realicé varias pruebas para comprobar su funcionamiento.

Probé distintos prompts para ver:

Si respondía correctamente.

Si generaba código funcional.

Si mantenía coherencia en las respuestas.

Por ejemplo, le pedí que generara una página web sobre animales, y el modelo generó directamente el código HTML correspondiente.

Aunque el modelo está optimizado para programación, comprobé que también puede mantener conversaciones básicas.

Estas pruebas confirmaron que:

El servidor local estaba funcionando.

El agente estaba correctamente integrado.

El modelo utilizaba la GPU para procesar las peticiones.

7. Conclusión

En este trabajo he implementado un agente de Inteligencia Artificial funcionando completamente en local, utilizando un modelo ligero optimizado para programación.

He configurado el entorno, optimizado el uso de la GPU, levantado un servidor local compatible con la API de OpenAI e integrado el sistema con OpenCode para crear un agente funcional.

Este proyecto demuestra que es posible ejecutar modelos avanzados sin depender de servicios en la nube, siempre que se optimicen correctamente los recursos disponibles.
