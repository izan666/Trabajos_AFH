1. ¿Qué es el Fine-Tuning?

El fine-tuning es un proceso mediante el cual se toma un modelo previamente entrenado y se ajusta con un conjunto de datos más específico para que mejore en una tarea concreta.

A diferencia del pre-entrenamiento (donde el modelo aprende patrones generales del lenguaje), el fine-tuning:

Especializa el modelo.

Mejora su rendimiento en un dominio específico.

Reduce errores en tareas concretas.

Por ejemplo, modelos desarrollados por OpenAI pueden ser ajustados posteriormente para tareas médicas, legales o técnicas.

En este trabajo, el objetivo fue adaptar un modelo base a un conjunto de datos específico para mejorar su comportamiento en una tarea concreta.

3. Preparación del entorno

Antes de comenzar el proceso de fine-tuning, preparé el entorno de trabajo:

Instalé las dependencias necesarias.

Verifiqué compatibilidad de librerías.

Preparé el dataset formateado correctamente.

Cargué el modelo base que iba a ajustar.

También comprobé que el hardware disponible fuera suficiente para realizar el ajuste sin errores de memoria.

4. Preparación del Dataset

Para poder hacer fine-tuning, primero tuve que preparar correctamente los datos.

En mi caso:

Organicé el dataset en formato adecuado (prompt-respuesta).

Limpié datos innecesarios.

Revisé errores de formato.

Convertí el archivo al formato requerido por la librería de entrenamiento.

El modelo necesita que los datos estén estructurados correctamente para poder aprender de ellos.

5. Configuración del Fine-Tuning

Una vez preparado el dataset, configuré los parámetros del entrenamiento.

Definí:

Número de épocas.

Tamaño de batch.

Learning rate.

Carpeta de salida.

Guardado automático de checkpoints.

También ajusté los parámetros para evitar sobrecargar la VRAM y evitar errores durante el entrenamiento.

En esta fase es muy importante encontrar un equilibrio:

Si el learning rate es demasiado alto, el modelo puede perder conocimiento previo.

Si es demasiado bajo, el aprendizaje será muy lento.

6. Ejecución del Fine-Tuning

Después de configurar todo correctamente, ejecuté el proceso de entrenamiento.

Durante esta fase:

El modelo procesa el dataset.

Ajusta sus pesos internos.

Minimiza la función de pérdida (loss).

Guarda checkpoints automáticamente.

El proceso puede tardar dependiendo del tamaño del modelo y del dataset.

Supervisé que:

No hubiera errores de memoria.

El loss descendiera progresivamente.

El entrenamiento no se interrumpiera.

7. Evaluación del Modelo Ajustado

Una vez finalizado el fine-tuning, probé el modelo con nuevos prompts para comprobar si realmente había mejorado.

Comparé:

Respuestas antes del fine-tuning.

Respuestas después del fine-tuning.

El modelo mostró respuestas más precisas y adaptadas al dominio específico con el que fue entrenado.
